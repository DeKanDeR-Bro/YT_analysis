# Методология сбора открытых данных (YouTube Data API v3)

Этот документ описывает технический процесс и этические принципы сбора данных для аналитического исследования "Minecraft War". Сбор данных осуществляется исключительно через официальный публичный интерфейс YouTube Data API v3.

## 1. Источник данных
Все данные получены из открытых источников с использованием официального API YouTube.
- **API:** Google YouTube Data API v3
- **Документация:** [https://developers.google.com/youtube/v3](https://developers.google.com/youtube/v3)
- **Тип данных:** Публичная мета-информация о каналах, видео и комментариях.

**ВАЖНО:** Мы НЕ собираем персональные данные пользователей, приватные сообщения, скрытые видео или любую информацию, не предназначенную для публичного просмотра.

## 2. Используемые методы API

### 2.1. Информация о канале (`channels.list`)
Используется для получения базовой статистики каналов.
- **Endpoint:** `GET https://www.googleapis.com/youtube/v3/channels`
- **Параметры:** `part=statistics,snippet`, `id={CHANNEL_ID}`
- **Собираемые поля:** 
  - Название канала (`snippet.title`)
  - Общее кол-во просмотров (`statistics.viewCount`)
  - Кол-во подписчиков (`statistics.subscriberCount`) - *только если не скрыто автором*.

### 2.2. Поиск видео (`search.list` / `playlistItems.list`)
Используется для составления списка видео каждого канала. Мы используем метод итерации по плейлисту "Uploads" (Загрузки), так как это наиболее эффективный способ получить все видео канала.
- **Endpoint:** `GET https://www.googleapis.com/youtube/v3/playlistItems`
- **Параметры:** `part=snippet`, `playlistId={UPLOADS_PLAYLIST_ID}`, `maxResults=50`
- **Собираемые поля:** ID видео, название, дата публикации.

### 2.3. Статистика видео (`videos.list`)
Используется для получения метрик вовлеченности конкретных видео.
- **Endpoint:** `GET https://www.googleapis.com/youtube/v3/videos`
- **Параметры:** `part=statistics,contentDetails`, `id={VIDEO_ID}`
- **Собираемые поля:**
  - Просмотры (`statistics.viewCount`)
  - Лайки (`statistics.likeCount`)
  - Комментарии (`statistics.commentCount`)
  - Длительность (`contentDetails.duration`)

### 2.4. Комментарии (`commentThreads.list`)
Используется для выборочного анализа активности аудитории.
- **Endpoint:** `GET https://www.googleapis.com/youtube/v3/commentThreads`
- **Параметры:** `part=snippet`, `videoId={VIDEO_ID}`, `order=relevance`
- **Собираемые поля:** Текст комментария, дата публикации. *Имена авторов анонимизируются или используются только для агрегированной статистики.*

## 3. Соблюдение квот и ограничений (Quotas)
Скрипт сбора данных (`collector.py`) спроектирован с учетом ограничений API:
- Стандартная квота: 10,000 единиц в день.
- Оптимизация: Использование batch-запросов (запрос статистики для 50 видео за один вызов) снижает потребление квоты в 50 раз.
- Паузы: При получении ошибки `403 Quota Exceeded` скрипт автоматически останавливает работу.

## 4. Воспроизводимость
Для воспроизведения сбора данных необходимо:
1. Получить собственный API Key в [Google Cloud Console](https://console.cloud.google.com/).
2. Создать файл `config.py` или переменную окружения `YOUTUBE_API_KEY`.
3. Запустить скрипт сбора.
